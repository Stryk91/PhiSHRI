‚óè Multi-Agent Coordination Architecture

  Problem

  Multiple AI agents (Claude in different contexts, IDE
  assistants, etc.) need to collaborate on the same codebase
   asynchronously without direct communication channels.

  Solution: Dual-Layer Messaging System

  Layer 1: JSONL Event Feed (Simple, Append-Only Log)

  File: phiwave/agent_feed.py ‚Üí writes to
  docs/agent-feed.jsonl

  - Purpose: Persistent activity log for async coordination
  - Format: One JSON object per line with {timestamp, agent,
   action, details}
  - API:
    - log_action(action, details, agent="...") - Log any
  event
    - read_tail(n) - Read last N events
    - notify_junie(message, ...) - Structured notifications
  between agents

  Use cases:
  - "IDE Claude finished Task 3" ‚Üí Terminal Claude sees it
  and assigns Task 4
  - "Found bug in engine.py:125" ‚Üí Other agents can review
  - Session history / audit trail

  Example:
  from phiwave.agent_feed import log_action

  log_action("task_complete", {
      "task": "Add authentication",
      "files": ["auth.py", "middleware.py"],
      "tests_passing": True
  })

  Layer 2: MCP Message Hub (Real-Time Database)

  File: mcp_agent_hub.py (FastMCP server with SQLite)

  - Purpose: Real-time agent-to-agent messaging with queries
  - Storage: SQLite database with message queue
  - Exposed as MCP tools that any Claude instance can call:
    - post_message(sender, content, msg_type) - Send a
  message
    - get_messages(limit, unread_only) - Fetch recent/unread
    - mark_processed(message_id) - ACK a message
    - get_agent_status() - See who's active
    - search_messages(query) - Search history

  Use cases:
  - Agent A: "I'm blocked on X, need help"
  - Agent B polls get_messages(unread_only=True), sees it,
  responds
  - Real-time coordination during active sessions

  Key Design Principles

  1. No direct coupling: Agents never call each other - they
   read/write shared state
  2. Async by default: JSONL append is instant, readers poll
   when convenient
  3. MCP integration: Tools available in Claude Code
  sessions automatically
  4. Human-readable: Both systems use JSON - easy to debug,
  grep, or manually inspect
  5. Persistence: JSONL = permanent record, SQLite =
  queryable recent activity

  Benefits for Multi-Agent Projects

  - ‚úÖ Handoffs: "I finished X, you do Y next"
  - ‚úÖ Progress tracking: See what all agents have done
  - ‚úÖ Conflict avoidance: "I'm working on auth.py" ‚Üí others
   avoid it
  - ‚úÖ Testing workflows: Dev agent ‚Üí Test agent ‚Üí Review
  agent
  - ‚úÖ Debugging: Full audit trail of who did what when

  Quick Setup for Your Project

  1. Copy agent_feed.py utility module
  2. Set up MCP server with mcp_agent_hub.py (optional but
  powerful)
  3. Configure Claude Code to auto-connect to MCP server
  4. Have agents log key events: log_action("task_complete",
   {...})
  5. Agents poll feed/hub to coordinate

  That's it - decentralized coordination without custom
  infrastructure.
